VOICE SYSTEM MODES
Add a mode called:
- ES3 Script Mode

In ES3 Script Mode, NOVA should follow the sequence below.

========================
1. SIGN ON FLOW
========================
When selector starts NOVA, begin the Jennifer Sign On flow.

NOVA says:
"Enter equipment ID."

Expected response:
equipment ID digits like:
0 0 0 1

Then NOVA says:
"Confirm equipment 0001."

Expected confirm:
- confirm
- yes
- affirmative

Expected deny:
- cancel
- no
- negative

Then NOVA says:
"Enter maximum pallet count for jack 0001."

Expected response:
numeric value, usually 2

Then NOVA says:
"Confirm maximum pallet count."

Expected confirm:
- confirm
- yes

If denied:
repeat the step until confirmed.

========================
2. SAFETY CHECKLIST
========================
After pallet count is confirmed, NOVA must read each safety item one by one:

- Brakes okay?
- Battery guard okay?
- Horn okay?
- Wheels okay?
- Hydraulics okay?
- Controls okay?
- Steering okay?
- Welds okay?
- Electric wiring okay?

For each item:
Confirm responses allowed:
- confirm
- yes
- affirmative

Deny responses allowed:
- cancel
- no
- negative

If denied:
- log failed safety item
- stop session
- show warning in UI
- do not continue to assignment flow

If all are confirmed:
continue to loading picks flow

========================
3. LOAD PICKS FLOW
========================
After safety checklist passes:

NOVA says:
"Say load picks."

Expected response:
- load picks

If anything else is said:
repeat:
"Say load picks."

After user says load picks:
NOVA loads only the assignment assigned to the selector user id.

Do not use the paper sample assignment.
Use the live assignment assigned to that selector.

NOVA then reads assignment summary in ES3 style:
- start aisle [start aisle] end aisle [end aisle]
- slots [number of stops]
- total cases [total cases]
- cube [total cube if available]
- pallets [total pallets]
- estimated goal [goal if available]
- to continue say ready

Expected response:
- ready

Then NOVA says:
- position alpha pallet get chep
Expected response:
- ready

Then NOVA says:
- position bravo pallet get chep
Expected response:
- ready

========================
4. PICK FLOW
========================
For each stop in the assignment, NOVA should behave like this:

NOVA says:
"New aisle [aisle] slot [slot]"

Selector must say the stop check digits.

If correct:
NOVA says:
"Grab [qty] [Alpha or Bravo if assigned]"

Examples:
- Grab 24 Alpha
- Grab 24 Bravo

STRICT CHECK LOGIC
- do not continue until the correct code is spoken
- if wrong, repeat what was heard and say invalid
- never freeze
- keep listening until correct

Examples:
- "You said 999. Invalid."
- "You said ready. Invalid."
- "No input received."

After valid stop confirmation:
NOVA reads grab quantity.

Expected follow-up can include:
- grab
- ready
- confirm

========================
5. BATCH COMPLETE FLOW
========================
After last stop:

NOVA says:
"Batch complete deliver pallet bravo to door [door number]"

Selector confirms door code.
Use live door code from assignment, not the paper sample.

Then NOVA says:
"Apply labels to pallet alpha"

Selector confirms alpha label digits or says alpha

Then NOVA says:
"Apply labels to pallet bravo"

Selector confirms bravo label digits or says bravo

Then NOVA says:
"Deliver bravo pallet to door [door number]"

Selector confirms staging code

Then NOVA says:
"Deliver alpha pallet to door [door number]"

Selector confirms staging code again

Then NOVA says:
"Say load picks"

If user says load picks:
start next assigned batch if available

========================
6. CONFIRM / DENY COMMANDS
========================
Anywhere confirm/deny is needed, allow these sets:

CONFIRM:
- yes
- confirm
- affirmative

DENY:
- no
- cancel
- negative

Normalize all speech input to lowercase before matching.

========================
7. ES3 VOICE COMMAND LIST
========================
Add these supported commands to NOVA:

HELP MODE COMMANDS
- repeat labels
- makeup skips
- get drops
- close batch
- go to aisle [number]

BASIC PICKING COMMANDS
- load picks
- base items
- confirm
- no
- previous
- base complete
- grab
- remove pallet
- skip slot
- report damage
- damage [number]
- slot empty
- recap weights
- next
- previous
- exit list
- no labels

Each command must map to an internal action.

Command behavior:
- repeat labels -> replay label instructions
- makeup skips -> retrieve skipped slots
- get drops -> retrieve replenishment drops
- close batch -> remove from current assignment / close batch state
- go to aisle [number] -> jump navigation helper
- load picks -> load assigned batch
- base items -> review base items
- confirm -> confirm current prompt
- no -> deny current prompt
- previous -> replay previous item or weight item
- base complete -> mark base setup done
- grab -> confirm case quantity action
- remove pallet -> mark pallet removed from pick slot
- skip slot -> mark slot skipped
- report damage -> start damage flow
- damage [number] -> report damaged case count
- slot empty -> mark slot empty / trigger replenishment need
- recap weights -> recap random weight entries
- next -> next weight/list item
- exit list -> exit list mode
- no labels -> label print failure flow
